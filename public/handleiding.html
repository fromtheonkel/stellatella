01

Wat is Stellatella?

Stellatella is een interactieve webshop voor chocolade. Bezoekers kunnen producten bekijken, een account aanmaken, chocolade in hun winkelwagen plaatsen en een bestelling afronden. Daarnaast is er een informatiepagina over hoe chocolade gemaakt wordt en een "Over Mij" pagina.

Belangrijk om te weten: een website bestaat uit twee delen. De frontend is wat je ziet in de browser (knoppen, tekst, kleuren). De backend is wat er achter de schermen gebeurt (data opslaan, inloggen controleren).

Welke technologieen zijn gebruikt?

De website is gebouwd met moderne webontwikkeling tools. Hier is een overzicht van elke technologie en wat die doet:

Technologie Wat doet het?
Next.js 15 Het framework (geraamte) van de website. Next.js maakt het makkelijk om pagina's te bouwen die snel laden.
React Een library om de interface op te bouwen uit herbruikbare "componenten" (blokken).
TypeScript Een uitbreiding op JavaScript die helpt fouten te voorkomen door types toe te voegen aan code.
Tailwind CSS Een styling systeem waarmee je direct in je HTML kleuren, marges en layouts toevoegt via class-namen.
Neon PostgreSQL De online database waar alle producten, gebruikers en bestellingen worden opgeslagen.
bcryptjs Een library die wachtwoorden veilig versleutelt (hashed), zodat ze niet leesbaar zijn in de database.
Vercel Het platform waar de website online staat (hosting). Bij elke update wordt de site automatisch opnieuw gebouwd.
GitHub Hier staat alle code opgeslagen. Werkt samen met Vercel: push je code, dan gaat de site automatisch live.

"Next.js combineert de voorkant en achterkant in een project, zodat je niet twee aparte systemen hoeft te bouwen."

De voorkant: wat de gebruiker ziet

Componenten

De website is opgebouwd uit componenten — herbruikbare bouwblokken. De Header (navigatiebalk) is bijvoorbeeld een component die op elke pagina verschijnt. Je schrijft de code een keer en gebruikt hem overal.

// ProductCard.tsx — Een kaartje voor elk product export default function ProductCard({ naam, prijs }) { return ( <div> <h3>{naam}</h3> <p>Prijs: {prijs}</p> </div> ); }

Styling met Tailwind CSS

In plaats van een apart CSS-bestand schrijf je stijlen direct als class-namen in de HTML. Tailwind biedt duizenden kleine klassen die je combineert:

<!-- Traditionele CSS --> <button class="mijn-knop">Klik</button> /* In apart CSS bestand: .mijn-knop { background: brown; color: white; } */ <!-- Met Tailwind CSS --> <button class="bg-[#8B4513] text-white px-8 py-3 rounded">Klik</button> /* Geen apart bestand nodig! */

De achterkant: Server Actions

Normaal heb je een aparte backend server nodig om formulieren te verwerken. Next.js heeft Server Actions: functies die op de server draaien maar direct vanuit je formulier aangeroepen kunnen worden.

// In actions.ts 'use server'; export async function loginAction(formData) { const naam = formData.get('gebruikersnaam'); const wachtwoord = formData.get('wachtwoord'); // Zoek gebruiker in database // Controleer wachtwoord // Maak sessie aan }
<!-- In login/page.tsx --> <form action={loginAction}> <input name="gebruikersnaam" /> <input name="wachtwoord" type="password" /> <button>Inloggen</button> </form>

Hoe werkt het? Wanneer je op "Inloggen" klikt, stuurt het formulier de data naar de loginAction functie. Die draait op de server, checkt de database, en stuurt je door naar de juiste pagina. Alles zonder een aparte API te bouwen!

De database

Alle data wordt opgeslagen in een PostgreSQL database die draait bij Neon (een cloud database service). De website maakt verbinding met de database via een DATABASE_URL — een soort adres dat in een geheim bestand staat.

Tabellen

De database heeft 4 tabellen (denk aan Excel-sheets):

TabelWat slaat het op?Voorbeeldvelden
st_gebruikers Alle accounts id, gebruikersnaam, email, hashedwachtwoord
st_producten Alle chocoladeproducten id, naam, beschrijving, prijs, afbeelding, categorie
st_bestellingen Geplaatste bestellingen id, gebruiker_id, totaal, datum
st_bestelregels Producten per bestelling id, bestelling_id, product_id, aantal, prijs

Hoe vraag je data op?

// Alle producten ophalen uit de database const producten = await sql` SELECT * FROM st_producten ORDER BY naam `; // Een specifiek product ophalen const product = await sql` SELECT * FROM st_producten WHERE id = ${productId} `;

Waarom "st_"? Alle tabellen beginnen met st_ (Stellatella). Dit is om ze te onderscheiden van andere tabellen in dezelfde database die voor andere projecten worden gebruikt.

07

Inloggen en accounts

Het inlogsysteem werkt met cookies — kleine stukjes data die je browser onthoudt. Wanneer je inlogt, wordt er een cookie opgeslagen met je gebruikers-ID. Bij elk volgend bezoek weet de website zo wie je bent.

Het inlogproces stap voor stap

1

Gebruiker vult naam en wachtwoord in

2

Server zoekt de gebruiker in de database

3

Wachtwoord wordt vergeleken met de hash in de database (bcrypt)

4

Als het klopt: cookie wordt aangemaakt met de gebruikers-ID

5

Gebruiker wordt doorgestuurd naar de homepagina

Wachtwoord hashing: Wachtwoorden worden nooit als platte tekst opgeslagen. Ze worden omgezet in een onleesbare reeks tekens (een "hash"). Zelfs als iemand de database zou hacken, kan die de wachtwoorden niet lezen.

De winkelwagen

De winkelwagen wordt opgeslagen in een cookie — niet in de database. Dit betekent dat je ook zonder account producten kunt toevoegen. De cookie bevat een lijstje met product-ID's en aantallen.

// Zo ziet de winkelwagen cookie eruit (JSON) [ { "productId": 1, "aantal": 2 }, { "productId": 5, "aantal": 1 } ] // = 2x Klassieke Pralines + 1x Karamel Zeezout Bonbons

Animaties en interactie

De website gebruikt twee soorten animaties om de pagina's levendiger te maken:

1. Scroll-animaties (IntersectionObserver)

Elementen verschijnen met een vloeiende animatie wanneer je ernaartoe scrollt. Dit werkt met de IntersectionObserver API — een browser-functie die detecteert wanneer een element in beeld komt.

// Detecteer wanneer een element zichtbaar wordt const observer = new IntersectionObserver(([entry]) => { if (entry.isIntersecting) { setVisible(true); // Element is in beeld! } }); // Het element begint onzichtbaar en schuift omhoog style={{ opacity: visible ? 1 : 0, transform: visible ? 'translateY(0)' : 'translateY(30px)', transition: 'all 0.7s ease' }}

2. Hover-effecten (CSS Transitions)

Knoppen en kaarten reageren wanneer je er met de muis overheen gaat. Dit zijn pure CSS-effecten met transition:

/* Product kaart gaat omhoog + schaduw bij hover */ className="hover:-translate-y-1 hover:shadow-xl transition-all" /* Foto zoomt in bij hover */ className="group-hover:scale-110 transition-transform duration-500"

Wat zijn Tailwind prefixes? In Tailwind voeg je hover: toe voor hover-effecten, sm: voor tablet/desktop stijlen, en group-hover: om een kind-element te animeren wanneer je over het ouder-element hovert.

10

Online zetten met Vercel

De website wordt gehost op Vercel, het platform dat Next.js heeft gemaakt. Het deployment proces is volledig automatisch:

Stap 1

Code wordt geschreven en getest op de eigen computer (localhost)

Stap 2

Wijzigingen worden ge-commit en ge-pushed naar GitHub

Stap 3

Vercel detecteert de push en bouwt de website automatisch opnieuw

Stap 4

Binnen ~1 minuut staat de nieuwe versie live op het internet!

# Code opslaan en publiceren git add . git commit -m "Nieuwe functie toegevoegd" git push # Dat is alles! Vercel doet de rest automatisch.

Environment Variables

Geheime instellingen zoals de DATABASE_URL staan niet in de code (dat zou onveilig zijn). Ze worden apart ingesteld in het Vercel dashboard als Environment Variables. Lokaal staan ze in een .env.local bestand dat niet naar GitHub wordt geupload.
